6. **The cross-page entity backlog is now complete for the property-tab family + portfolio.** Phase 5 (entity-wiring sprint) can begin with high confidence in the priorities.

## How this file is regenerated

Every time `/audit-page-datapoints` runs (whether on a new route or as a re-audit on an existing one), the skill:

1. Reads every `plan.md` under `.claude/data-audit/pages/*/plan.md`
2. Aggregates each plan's `## 3. Entity Backlog` section — entities, surfaces unlocked per page, statuses
3. Sums `Surfaces unlocked` across pages per entity
4. Re-sorts by total surfaces unlocked (desc)
5. Rewrites this file in place

This means the file is **always derived** from the per-page plans. If you want to fix or refine an entity row, edit the corresponding `plan.md` Entity Backlog section, then re-run `/audit-page-datapoints` against that route. Editing `pages/INDEX.md` directly will be overwritten on next run.
