# Impeccable Design Context — Valgate

## Design Context

### Users
Individual landlords managing their own property portfolios. They use the app on desktop during business hours — checking rental income, reviewing analytics, tracking occupancy, managing documents. They're not developers or data scientists, but they're comfortable with structured software. They want to feel in control, not overwhelmed.

### Brand Personality
**Confident, modern, sharp.**

Valgate should feel like the kind of tool a serious person uses. Not flashy, not casual. Clean lines, deliberate choices, zero wasted chrome. The product's complexity (multi-property data, charts, financials) should feel effortlessly organized — not because the UI is simplified, but because the hierarchy is excellent.

Reference: Airbnb — airy, generous whitespace, photography-first clarity, refined sans typography, no decoration that doesn't earn its place. Premium without being cold.

Anti-references: Bloomberg Terminal (too dense/technical), generic SaaS dashboards with icon-heavy nav and color-coded KPI tiles.

### Aesthetic Direction
**Light mode primary.** The users are working in daylight, on a desktop, during business hours — light mode is correct here. Dark mode is optional but light is the experience to optimize for.

**Tone:** Clarity and calm. Complex property data should feel simple and manageable. Never let the interface feel like it's showing off. The data is the hero — the UI steps back.

**Visual character:**
- Generous, asymmetric spacing — not everything in cards, not everything centered
- Restrained use of blue (#004ac6 is the brand blue; #2563EB is the interactive/primary blue — keep both rare and purposeful)
- Neutrals tinted subtly toward the brand blue (no pure gray, no pure white)
- Typography does the heavy lifting; icons are supporting cast only
- Animations: purposeful, never decorative. State transitions and feedback only.
- Borders over shadows — 1px borders for separation; reserve elevation for overlays and modals only

### Typography Notes
The codebase currently uses Geist (body) and Bricolage Grotesque (display). **Bricolage Grotesque is on the reflex reject list** — avoid introducing it in new design work. For display headings on new screens, prefer a grotesque with more character and less ubiquity. Candidates worth evaluating: Neue Montreal, Satoshi, Aktiv Grotesk, or an appropriate Google Fonts alternative that fits "confident, modern, sharp" without being overused. Body text: Geist is acceptable for a product UI; it's clean and legible.

### Design Principles

1. **Hierarchy over decoration.** Every visual decision should serve information hierarchy. If an element doesn't help the user find what they need faster, it shouldn't exist.

2. **Space is intentional.** Generous whitespace isn't emptiness — it's the primary way the interface communicates calm and control. Vary spacing to create rhythm; don't apply the same padding everywhere.

3. **Blue is precious.** Brand blue appears only on interactive elements and primary actions. The moment it appears everywhere, it appears nowhere. Guard it.

4. **Data is the hero.** The property data — numbers, charts, addresses, dates — is what users came for. The UI's job is to frame it without competing with it. No decorative gradients, no large icon-in-box headers, no chrome that doesn't add information.

5. **Motion earns its place.** Animations exist only to communicate state (submission, loading, success, error) or smooth a jarring transition. Nothing animates for visual interest alone.

6. **Cards only when content is truly distinct and actionable.** Not every section needs a container. Use spacing and borders to create grouping. Never nest cards inside cards.

7. **Badge = metadata.** Supplementary labels (status, tags, recommendations) belong at the bottom of a card or inline with secondary text — not dominating the top of the visual hierarchy.

8. **Copy earns its space.** Every word should be specific and action-oriented. Avoid vague filler ("Your items will appear here") — say exactly what's true ("No drafts yet") and what to do next.

## Valgate Professional (Asset Manager POC)

### Users
Third-party asset managers who oversee multiple client portfolios simultaneously. Desktop-first, business hours. They jump between clients frequently and need parallel context — not a single-portfolio landlord view.

### Brand Personality
**Confident PM SaaS — calm, dense, organized.**

Reference: ClickUp / Linear / Wrike applied to asset management. Browser-like workspace tabs for client switching. Two tab layers must stay visually distinct (workspace tabs vs. in-client view tabs).

Anti-references: Generic admin dashboards with side-stripe alert accents, hero-metric gradient layouts, dark-mode-glow aesthetics.

### Aesthetic Direction
**Light mode primary** for Professional POC. Workspace tab strip: `#F4F5F7`. Active tab: white, merging into canvas with 2px top brand accent. Sidebar + canvas split below full-width tab bar.

### Design Principles (Professional-specific)

1. **Workspace tabs are the hero.** The tab bar must read instantly as "browser tabs for clients." Active tab elevated; inactive tabs recede.
2. **Sidebar syncs with active tab.** Highlighted client in sidebar must match active workspace tab.
3. **Scoped density.** Client portfolio pages breathe slightly more than cross-portfolio dashboard — one client's data, not everything at once.
4. **No side-stripe borders** on alerts or list items — use background tints and icons instead.
